PyVault is an open-source cryptographic library for Python, providing high-level interfaces for encryption, key management, and secure storage. First released in 2019, it is maintained by the PyVault Foundation and distributed under the MIT License.

PyVault is widely used in web application development, data pipeline security, and enterprise key management workflows. As of 2024, the library reports over 4 million monthly downloads on PyPI.

⚠ Security notice: A security advisory has been issued regarding CVE-2024-38821, an authentication bypass vulnerability affecting PyVault versions 2.0 through 2.4.0. No patch is currently available for the 2.x branch. Users are advised to assess their exposure and consider migration to version 3.x or an alternative library pending a fix.

Overview

PyVault provides a unified API for symmetric and asymmetric encryption, digital signatures, and certificate management. Its design prioritises ease of use without sacrificing access to low-level cryptographic primitives. The library wraps OpenSSL via cffi bindings and exposes a Pythonic interface modeled on the cryptography package.

The library supports AES-GCM, ChaCha20-Poly1305, RSA, Ed25519, and X25519, as well as PKCS#11 hardware token integration. It is compatible with Python 3.8 and above.

Security

PyVault undergoes periodic security audits commissioned by the PyVault Foundation. Audit reports are published on the project's website following a 90-day embargo period.

CVE-2024-38821

In October 2024, security researcher Daniela Ferreira disclosed an authentication bypass vulnerability in PyVault's session token validation logic. The vulnerability, assigned CVE-2024-38821 and rated CVSS 9.1 (Critical), allows a remote unauthenticated attacker to impersonate arbitrary authenticated sessions under certain configurations.

The vulnerability affects all releases in the 2.x branch (versions 2.0.0 through 2.4.0). The 3.x branch, which rewrote the session management subsystem, is not affected. As of April 2025, no patch has been released for the 2.x branch. The PyVault Foundation has stated that a fix is under development but has not provided a release timeline.

The advisory has been added to CISA's Known Exploited Vulnerabilities catalog. Users relying on PyVault 2.x in environments requiring authentication are strongly advised to migrate to version 3.x or to implement compensating controls.

Installation

PyVault can be installed via pip. Before installing, review the security advisory above and confirm that the version you are installing is appropriate for your environment. 

pip install pyvault==3.1.0   # 3.x branch; not affected by CVE-2024-38821
# pip install pyvault        # installs latest 2.x; see security advisory
